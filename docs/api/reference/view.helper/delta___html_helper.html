Class Delta_HTMLHelper

Description | Constants | Properties | Methods

Description

HTML の作成を補助するヘルパメソッドを提供します。 このヘルパは、$html という変数名であらかじめテンプレートにインスタンスが割り当てられています。
<?php echo $html->{method}; ?>
global_helpers.yml の設定例:
html: # ヘルパクラス名。 class: Delta_HTMLHelper # includeCSS() メソッドが参照する基底 CSS パス。 baseCSSPath: /assets/css # includeJS() メソッドが参照する基底 JavaScript パス。 baseJSPath: /assets/js # image() メソッドが参照する基底イメージパス。 baseImagePath: /assets/images # 圧縮した CSS のディレクトリを参照する場合に TRUE を指定。includeCSS() の項を参照。 cssMinify: FALSE # 圧縮した JS のディレクトリを参照する場合に TRUE を指定。includeJS() の項を参照。 jsMinify: FALSE
その他に指定可能な属性は Delta_Helper クラスを参照。

autoLink()
public string autoLink($string, $attributes = array(), $email = FALSE, $cushionPath = NULL)
文字列中に含まれる URI や E-Mail アドレスにアンカーを追加します。 このメソッドは、HTML ソースに対し実行するべきではありません。HTML タグに含まれるリンクに重ねてアンカーが張られる可能性があります。
Property Type Description
$string string
対象となる文字列。
$attributes mixed
タグに追加する属性。Delta_HTMLHelper::link() メソッドを参照。
$email bool
TRUE を指定した場合、E-Mail アドレスもリンク対象とする。
$cushionPath string
クッションパス。 文字列中に外部の URI が含まれる場合、直接アンカーを張るのではなく、クッションページに遷移させることが可能。 クッションページには遷移先の URI が '?uri={URI}' 形式で付加される。 指定可能なパスの書式は Delta_Router::buildRequestPath() メソッドを参照。
{return} string
変換後の文字列を返します。
  • Author:
    Naomichi Yamakita <naomichi.y@delta-framework.org>

buildAssetPath()
public void buildAssetPath($path, $type, $extra)
指定されたパスを元に、Web から参照可能なリクエストパスを生成します。
Property Type Description
$path string
変換対象のファイルパス。既定パスは type 値によります。 例えば CSS ファイル 'base.css' を指定した場合に生成されるリクエストパスは '/assets/css/base.css' となります。
$type string
path のファイルタイプ。指定可能な値は次の通り。
  • css: 'baseCSSPath' からの相対パスを生成。
  • js: 'baseJSPath' からの相対パスを生成。
  • image: 'baseImagePath' からの相対パスを生成。
$extra array
タグの出力オプション。
  • absolute: 相対パスを絶対パスに変換します。
  • query: path に追加するクエリパラメータ。連想配列形式で指定。
  • ignoreCache: ファイルの最終更新時刻をクエリに追加する場合は TRUE を指定。 ファイルがローカルキャッシュされる問題を回避したい場合に有効です。 生成されるパスは '/assets/css/example.css?1315552397' のような形式になります。
  • Author:
    Naomichi Yamakita <naomichi.y@delta-framework.org>
