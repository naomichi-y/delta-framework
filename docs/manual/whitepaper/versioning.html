Hacks

アプリケーションを共同で開発する際、CVS (Concurrent Versions System) や SVN (Subversion) といったバージョン管理システムは必要不可欠です。ここではアプリケーションを運用する上で、リポジトリ上で管理する必要のないファイル・ディレクトリについて解説します。

delta はクラスファイルやテンプレート、YAML といった設定ファイルの読み込みを高速化するため、アプリケーション内でキャッシュ・一時ファイルを作成します。これらのファイルはアプリケーションを実行するサーバ上に作成されていれば良いため、リポジトリで管理する必要はありません。これらのファイルはリポジトリへコミットせず、対象ディレクトリ自体をコミット拒否設定にしておくと良いでしょう。

次にリポジトリで管理する必要のないファイル・ディレクトリの一覧を示します。

管理対象外 説明
cache delta が生成するキャッシュファイルを管理。cache 内は更に autoload (クラスパスキャッシュ)、templates (テンプレートキャッシュ)、yaml (YAML キャッシュ) ディレクトリから構成されます。
data (任意) アプリケーションデータを格納。
tmp delta が生成する一時ファイルを管理します。
webroot/.htaccess ModRewrite のパスを設定。開発者の環境、またはサーバ環境の都合に合わせて場合により削除可能。 (.htaccess を使用せず、サーバ設定ファイルに ModRewrite の設定を書く場合も同様に配置不要です)

バージョン管理システムごとのファイル無視設定については各種マニュアルを参照して下さい。
