コアパッケージのディレクトリ構成

フレームワーク本体を含むコアパッケージには、クラスライブラリの他に各種ドキュメントやジェネレータ、ベンダープログラムを含んでいます。コアパッケージは Web から参照できる場所に配置するべきではありません。Linux 環境であれば /var/lib/delta などのディレクトリに配置することを推奨します。

下表はコアパッケージの主なディレクトリ・ファイル構成を示したものです。基本的に開発者が直接参照するディレクトリは docs ディレクトリのみとなるでしょう。

ディレクトリ構成 パーミッション 説明
command     各種ユーティリティコマンドを管理
delta   delta コマンドパス設定ファイル (Linux 用シェルスクリプト)
delta.bat   delta コマンドパス設定ファイル (Windows 用バッチファイル)
delta.php   delta コマンド本体
docs     各種ドキュメントを管理
api   API リファレンス
manual   マニュアル
libs   クラスライブラリ (API を参照)
skeleton   プロジェクト用スケルトンファイルを管理
vendors     フレームワークが使用するベンダーライブラリ
blowfish   blowfish 暗号ユーティリティ
spyc   YAML 解析ライブラリ
webapps   cpanel やサンプルアプリケーションを管理
VERSION   フレームワークのバージョンを記したファイル

プロジェクトのディレクトリ構成

プロジェクト構成ディレクトリは、開発者が作成したアクションやコンポーネントを管理します。

ディレクトリ構成 パーミッション 説明
console     コンソールアプリケーションを管理
commands   コマンドディレクトリ
deltac   コマンド起動スクリプト (Linux)
deltac.bat   コマンド起動スクリプト (Windows)
deltac.php   コマンド起動スクリプト
cache   775 delta (及びベンダーライブラリ) が生成する各種キャッシュを格納
file   クラスパスキャッシュディレクトリ
templates   テンプレートキャッシュディレクトリ
yaml   YAML キャッシュディレクトリ
config     アプリケーション設定情報を管理
base_dicon.yml   DI コンテナ設定ファイル
global_behaviors.yml   ビヘイビア設定ファイル
application.yml   アプリケーション設定ファイル
global_helpers.yml   ヘルパ設定ファイル
delta_env.php   環境設定ファイル
routes.yml   ルータ設定ファイル
site.yml   サイト設定ファイル
data     アプリケーションが利用する各種データを管理 (data ディレクトリはユーザのプライベート写真など、web 公開ディレクトリに配置すべきではないデータを管理するためのディレクトリです)
templates     差し込みメールテンプレートやメンテナンステンプレートなど、アプリケーション全体から利用する可能性があるファイルを管理
html fatal_error.php   致命的なエラーが発生した際に出力するテンプレート
http_error.php   HTTP ステータスエラーを出力するテンプレート

 

maintenance.php

   メンテナンス画面を出力するテンプレート
skeleton.php   delta add-action コマンドで使用される HTML テンプレート
system_error.php   例外が発生した際に出力するテンプレート (Delta_ExceptionOutputDelegate クラスで使用される)
mail     メールテンプレート設置ディレクトリ。Delta_MailSender::setTemplate() メソッドで参照される
theme     テーマディレクトリ。詳しくは テーマ機能 を参照
libs     アプリケーションクラスライブラリ。開発者が作成した共有コンポーネントを配置する
dao   Delta_DAO を継承した DAO クラスを管理
entity   Delta_Entity を継承したエンティティクラスを管理
service   Delta_Service を継承したサービスクラスを管理
logs   775 delta (及びベンダーライブラリ) が生成する各種ログを格納
modules {module}(+)     モジュールディレクトリ
actions   アクションクラスディレクトリ
behaviors   ビヘイビアディレクトリ
config   モジュール設定ディレクトリ
templates   テンプレートディレクトリ
tmp 775 一時生成ファイル格納ディレクトリ
vendors   アプリケーションが必要とする外部ライブラリを格納
webroot     Web 公開ディレクトリ
index.php   Web アプリケーション起動スクリプト

表内の (+) は、対象属性が複数回指定可能なことを表します。
